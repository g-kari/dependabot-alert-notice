# Git 運用方針

## コミット前の必須チェック

```bash
just test    # go test ./... -v -race -count=1
just lint    # go vet ./...
```

両方がパスしてからコミットする。

## コミットルール

- コミットメッセージは日本語
- 機能単位でコミット（大きな変更は適切な単位に分割）
- テストと実装は同一コミットに含める（テストだけ、実装だけは原則NG）

## コミットメッセージ例

```
feat: Dependabotアラートの手動取得トリガー追加
fix: 除外リポジトリ追加時に既存アラートを一覧から除外
test: github.Client のモックテスト追加
refactor: store パッケージのロック粒度を改善
```
